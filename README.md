# 3. บทนำ

ปัจจุบันการซื้อสินค้าแฟชั่นผ่านช่องทางออนไลน์ได้รับความนิยมเพิ่มขึ้นอย่างต่อเนื่อง โดยเฉพาะเครื่องประดับแฟชั่น เช่น แหวน สร้อยคอ ต่างหู กำไล และสร้อยข้อมือ ซึ่งผู้บริโภคสามารถเลือกซื้อสินค้าได้สะดวกโดยไม่จำเป็นต้องเดินทางไปยังหน้าร้าน อย่างไรก็ตาม ร้านค้าขนาดเล็กจำนวนมากยังคงบริหารจัดการคำสั่งซื้อผ่านการแชทหรือการบันทึกข้อมูลด้วยตนเอง ส่งผลให้เกิดปัญหาในการติดตามคำสั่งซื้อ การตรวจสอบจำนวนสินค้าคงเหลือ และการสรุปยอดขาย

โครงงานนี้จึงมุ่งพัฒนาเว็บแอปพลิเคชันสำหรับซื้อขายเครื่องประดับแฟชั่นออนไลน์ เพื่อช่วยให้การบริหารจัดการร้านค้าเป็นระบบมากขึ้น รองรับการใช้งานของลูกค้า พนักงาน และผู้ดูแลระบบ โดยครอบคลุมกระบวนการตั้งแต่การเลือกสินค้า การสั่งซื้อ การตรวจสอบการชำระเงิน การจัดการสต็อก ไปจนถึงการจัดทำรายงานสรุปยอดขาย เพื่อเพิ่มประสิทธิภาพในการดำเนินงานและสร้างประสบการณ์ที่ดีให้กับผู้ใช้งาน

# 4. วัตถุประสงค์

1. พัฒนาเว็บแอปพลิเคชันสำหรับซื้อขายเครื่องประดับแฟชั่นออนไลน์
2. ออกแบบระบบให้รองรับการจัดการสินค้าที่มีหลายตัวเลือก เช่น สี ขนาด และวัสดุ
3. ลดความผิดพลาดในการจัดการคำสั่งซื้อและการควบคุมสต็อกสินค้า
4. พัฒนาระบบรายงานเพื่อช่วยให้ผู้ดูแลระบบสามารถติดตามยอดขายและข้อมูลสำคัญได้สะดวก
5. ประยุกต์ใช้ความรู้ด้านการวิเคราะห์ ออกแบบ และพัฒนาระบบสารสนเทศในการสร้างระบบจริง

---

# 5. ขอบเขตของระบบ

## 5.1 ส่วนของลูกค้า (Customer)

- สมัครสมาชิกและเข้าสู่ระบบ
- แก้ไขข้อมูลส่วนตัวและที่อยู่จัดส่ง
- ค้นหาและกรองสินค้าตามหมวดหมู่ ราคา สี หรือวัสดุ
- ดูรายละเอียดสินค้าและเลือกตัวเลือกของสินค้า
- เพิ่มสินค้าเข้าสู่ตะกร้า
- ใช้โค้ดส่วนลดและเลือกบริการห่อของขวัญ
- ชำระเงินโดยอัปโหลดหลักฐานการโอนเงิน
- ติดตามสถานะคำสั่งซื้อ
- ดูประวัติการสั่งซื้อ
- รีวิวและให้คะแนนสินค้าที่เคยซื้อ

## 5.2 ส่วนของพนักงาน (Staff)

- เข้าสู่ระบบพนักงาน
- ตรวจสอบคำสั่งซื้อใหม่
- ตรวจสอบหลักฐานการชำระเงิน
- ยืนยันคำสั่งซื้อ
- จัดการข้อมูลสินค้า
- จัดการสต็อกสินค้า
- อัปเดตสถานะการจัดส่ง
- บันทึกเลขพัสดุ

## 5.3 ส่วนของแอดมิน (Admin)

- เข้าสู่ระบบผู้ดูแลระบบ
- ดูภาพรวมของระบบ
- ดูรายงานยอดขาย
- ดูรายงานสินค้าคงเหลือ
- ดูรายงานผลประกอบการ
- ติดตามข้อมูลสถิติที่เกี่ยวข้องกับธุรกิจ

## 5.4 นอกขอบเขตของระบบ (Out of Scope)

ระบบจะไม่พัฒนาการเชื่อมต่อกับ Payment Gateway จริง ระบบสะสมแต้มสมาชิก ระบบคืนสินค้า การเชื่อมต่อบริษัทขนส่ง และการแจ้งเตือนผ่าน SMS โดยจะใช้ข้อมูลจำลองเพื่อสาธิตการทำงานของระบบ

# 9. ขั้นตอนการทำงานของระบบ

1. ลูกค้าสมัครสมาชิกหรือเข้าสู่ระบบ
2. เลือกสินค้าและเพิ่มลงในตะกร้า
3. กรอกข้อมูลการจัดส่งและยืนยันคำสั่งซื้อ
4. อัปโหลดหลักฐานการชำระเงิน
5. ระบบบันทึกคำสั่งซื้อและแจ้งเตือนพนักงาน
6. พนักงานตรวจสอบหลักฐานการชำระเงิน
7. พนักงานยืนยันคำสั่งซื้อ
8. ระบบตัดจำนวนสินค้าออกจากสต็อก
9. พนักงานจัดเตรียมสินค้าและบันทึกเลขพัสดุ
10. ลูกค้าติดตามสถานะการจัดส่ง
11. ลูกค้ารีวิวและให้คะแนนสินค้า
12. ระบบรวบรวมข้อมูลและสร้างรายงานให้ผู้ดูแลระบบ

### กรณีพิเศษ

- หากลูกค้าไม่ชำระเงินภายในเวลาที่กำหนด ระบบจะยกเลิกคำสั่งซื้อและคืนสินค้าเข้าสู่สต็อก
- หากตรวจสอบพบว่าหลักฐานการชำระเงินไม่ถูกต้อง พนักงานสามารถปฏิเสธคำสั่งซื้อและแจ้งให้ลูกค้าดำเนินการใหม่

# 10. ผลที่คาดว่าจะได้รับ

1. ได้เว็บแอปพลิเคชันซื้อขายสินค้าออนไลน์ที่สามารถใช้งานได้จริง
2. ลดระยะเวลาและข้อผิดพลาดในการจัดการคำสั่งซื้อ
3. เพิ่มความสะดวกให้กับลูกค้าในการเลือกซื้อสินค้าและติดตามสถานะ
4. ช่วยให้ผู้ดูแลระบบสามารถตรวจสอบข้อมูลยอดขายและรายงานต่าง ๆ ได้อย่างรวดเร็ว
5. ผู้พัฒนาได้รับประสบการณ์ในการวิเคราะห์ ออกแบบ และพัฒนาระบบสารสนเทศแบบครบวงจร
